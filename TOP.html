The Internet

I have a newwork with a router -> router connects to modem -> modem runs through telephone lines, turning network information into information mangeable by telephone infrastructure (and vice versa)
ISP = company that manages special routers than can access other ISPs routers
computer -> router -> modem -> telephone line -> ISP (mine) -> ISP (other) -> telephone line -> their modem -> their router -> their computer with info (server)

Command Line Basics:

  • $pwd where am I
  • $ls list files
    • -a all files including hidden
    -l long form (with info)
    -lh readable sizes
    -lhs readable sizes plus sort by size
    -lt reverse sort - can be used with others
  • $ln hard link
    • -s symbolic link (symlink)
  • $cd
    • takes me home $cd .. navigate up
  • $mkdir creates directory
    • -p creates nested directories
    -v print the results of mkdir
  • $cp copy a to b (b is a directory or file) OR
    • multiple copies a b to c (c is a directory>
    -v print the results of cp
    -Rv copy directory contents to another directory
    -f force overwrite
    -i confirm overwrite?
  • $rm delete file
  • $mv move file
  • | pipeline command to chain commands
  • $rmdir remove directory if empty
  • $rm -r remove directory if not empty
  • tab completion in command line
  • a flag is something like -a or -v
  • a directory is a file folder
  • arguments are a file (.txt) or a directory

Basic HTML

  • A tag: <p>
  • An element: <p>My butt stinks</p> element
  • Coder comment: <!--comment-->
  • div element (division) is a general purpose container for other elements
  • Add image/photo/pic: <img src="computer/photos/example.jpg" alt="An example for my notes, no image selected">
  • Link to external page: anchor element <a href="example.com" target="_blank">example.com</a> blank target will open external page in a new tab
  • Link to internal section on a webpage: <a href="#knowledge-check">Knowledge Check</a>
    • <h3 id="knowledge-check">Knowledge Check</h3>
    • href attribute should be #whatever-id-you-want-to-use and id attribute should be added to whichever element you want to jump to, using the same #whatever-id-you-want-to-use
  • Dead link: if leaving an href attribute blank to add a future link, use # <a href="#">Empty Link</a>
  • To make an image a link: <a href="#"><img src="examplephoto.com/example" alt="example"></a>
  • Lists: <li>around entire list</li>
    • <ol>for an ordered (numbered) list item</ol>
    • <ul>for an unordered (bulleted) list</ul>
  • Form: <form action="url">input and buttons go here</form>
    • form action could point to another web site (example.com/example.html) or a file within a website (example.html)
  • Inputs: <input type="text" placeholder="placeholder text">
    • This would create a one-line text box, for example, used for username/password which would be the placeholder text
    • can make an input required <input type="text" required>
  • Button: <button type="submit">this button would submit username/password</button>
  • Radio button: <label for="option1"> <input id="option1" type="radio" name="option1-option2">Option1</label> <label for="option2"> <input id="option2" type="radio" name="option1-option2">Option2</label>
    • Each button should be under it's own label and all related buttons should have the same name to create a group
    • By creating the group, selecting one radio button will deselect other radio buttons in the group (difference between button to select one thing and buttons to select multiple like check all that apply)
    • To be automatically checked (like country would auto choose USA): <input type="radio" name="option1-option2 checked>
  • Checkboxes: follows the same rules as the radio buttons <label for="option1"><input id="option1" type="checkbox" name="options">Option1</label><label for="option2"><input id="option2" type="checkbox" name="options">Option2</label>
