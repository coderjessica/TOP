The Internet

I have a network with a router -> router connects to modem -> modem runs through telephone lines, turning network information into information mangeable by telephone infrastructure (and vice versa)
ISP = company that manages special routers than can access other ISPs routers
computer -> router -> modem -> telephone line -> ISP (mine) -> ISP (other) -> telephone line -> their modem -> their router -> their computer with info (server)
server = special computer connected directly to the internet used to store webpages

Command Line Basics:

  • $pwd where am I
  • $ls list files
    • -a all files including hidden
    -l long form (with info)
    -lh readable sizes
    -lhs readable sizes plus sort by size
    -lt reverse sort - can be used with others
  • $ln hard link
    • -s symbolic link (symlink)
  • $cd
    • takes me home $cd .. navigate up
  • $mkdir creates directory
    • -p creates nested directories
    -v print the results of mkdir
  • $cp copy a to b (b is a directory or file) OR
    • multiple copies a b to c (c is a directory>
    -v print the results of cp
    -Rv copy directory contents to another directory
    -f force overwrite
    -i confirm overwrite?
  • $rm delete file
  • $mv move file
  • | pipeline command to chain commands
  • $rmdir remove directory if empty
  • $rm -r remove directory if not empty
  • tab completion in command line
  • a flag is something like -a or -v
  • a directory is a file folder
  • arguments are a file (.txt) or a directory

Git Basics

Really important info: When saving from local file to GitHub respository, make sure to enter the file of the respository you cloned. For example, if you are in coderjessica/TOP and that is where you clone your respository named "TOP", you now need to enter into coderjessica/TOP/TOP to add and push files to the remote repository.

How to Update GitHub from Local File

  1. check git status, see changes not staged for commit, add with "git add ." or "git add example.html" - should be green in git status
  2. git commit -m "notes on what I changed - git status will now say your branch is ahead by one and working tree is clean
  3. git push origin main then git status should say branch is up to date and working tree clean
  4. check github

HTML and CSS Basics

This is the section where I completed freeCodeCamp's interactive tutorial "Responsive Web Design" which I learned, honestly, way too much to put here. Good thing I have Google.

- Basic HTML

  • A tag: <p>
  • An element: <p>My butt stinks</p> element
  • Coder comment: <!--comment-->
  • div element (division) is a general purpose container for other elements
  • Add image/photo/pic: <img src="computer/photos/example.jpg" alt="An example for my notes, no image selected">
    • Note: alt tag is unnecessary for decorative images like background, it is used for accessibility
  • Link to external page: anchor element <a href="example.com" target="_blank">example.com</a> blank target will open external page in a new tab
  • Link to internal section on a webpage: <a href="#knowledge-check">Knowledge Check</a>
    • <h3 id="knowledge-check">Knowledge Check</h3>
    • href attribute should be #whatever-id-you-want-to-use and id attribute should be added to whichever element you want to jump to, using the same #whatever-id-you-want-to-use
  • Dead link: if leaving an href attribute blank to add a future link, use # <a href="#">Empty Link</a>
  • To make an image a link: <a href="#"><img src="examplephoto.com/example" alt="example"></a>
  • Lists: <li>around entire list</li>
    • <ol>for an ordered (numbered) list item</ol>
    • <ul>for an unordered (bulleted) list</ul>
  • Form: <form action="url">input and buttons go here</form>
    • form action could point to another web site (example.com/example.html) or a file within a website (example.html)
  • Inputs: <input type="text" placeholder="placeholder text">
    • This would create a one-line text box, for example, used for username/password which would be the placeholder text
    • can make an input required <input type="text" required>
  • Button: <button type="submit">this button would submit username/password</button>
  • Radio button: <label for="option1"> <input id="option1" type="radio" name="option1-option2">Option1</label> <label for="option2"> <input id="option2" type="radio" name="option1-option2">Option2</label>
    • Each button should be under it's own label and all related buttons should have the same name to create a group
    • By creating the group, selecting one radio button will deselect other radio buttons in the group (difference between button to select one thing and buttons to select multiple like check all that apply)
    • To be automatically checked (like country would auto choose USA): <input type="radio" name="option1-option2 checked>
  • Checkboxes: follows the same rules as the radio buttons <label for="option1"><input id="option1" type="checkbox" name="options">Option1</label><label for="option2"><input id="option2" type="checkbox" name="options">Option2</label>

- Applied Accessibility

  • Use alt text on images
    • Don't use alt text on decorative images like a background
  • Use headings to show hierarchical relationships of content, not just to make font bigger (do that with CSS, ya lazy!)
  • Main element: the main element is used to wrap around the main content. When users are using accessibility features, they can jump right to this information.
  • <div> groups content, <section> groups related content, <article> groups independent, self-containted content
  • Use <nav> to wrap the main navigation links
  • Use <header> and <footer> for accessibility
  • Use <audio id="examplename" controls> to wrap audio for accessibility
  • Use <figure> and <figcaption> for figures such as an image or chart.
  • Use <fieldset> around a form like a set of radio buttons for accessibility and wrap the title/question in <legend>
  • Date picker: <input type="date" id="pickdate" name="date"> along with < for="pickdate></label> (second one goes above the first one)
  • Use <time datetime="2021-01-31">last Wednesday OR Wednesday, January 1</time> for accessibility
  • Use CSS to hide a chart made for a screenreader only
    • .sr-only {position: absolute; left: -10000px; width: 1px; height: 1px; top: auto; overflow: hidden;}

- Responsive Web Design Principles

  • If viewing on a different device like phone: @media (max-width OR min-height: 300px) {p {font-size:}}
  • Use CSS codes max-width: 100%; and hieght: auto; to make sure an imagine is never wider than the container is is in and keep its aspect ratio
  • The simplest way to make your images properly appear on HR displays, is to use half the pixels for width and height of the image. For example, a 500x500 image should use height: 100px; and width: 100px;
  • Viewport units are relative to the viewport dimensions of a device: vw (width), vh (vh), vmin (minimum), vmax (maximum). body {width: 30vw;} would be 30% of the viewport's width.
